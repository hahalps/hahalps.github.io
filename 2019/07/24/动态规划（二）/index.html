1,整数划分

这道题的话还可以用母函数实现。

思路

  • 可以把整数划分问题变为恰好装满背包的完全背包问题；

  • 状态表示：f[i][j]用前i个物品恰好装满容量为j的背包的方案数；

  • 状态转移：f[i][j] = f[i-1][j-1] + f[i][j-i]

另一种思考方式：

  • 这个是我第一次学习整数划分时所遇到的解题思路；

  • 状态表示：f[i][j]表示和为i可以拆分成j个数的方案数；

  • 状态转移：f[i][j] = f[i-1][j-1] + f[i-j][j]

  • 解释一下状态的转移：针对于f[i][j]来说，其结果可以划分成两个集合，一个是包含最小值为1的方案，另一个集合所有方案里的值都大于1，这样总的结果就等于两个集合方案数的和。前者对应的是f[i-1][j-1],把1加进去的话就变为f[i][j]，且里面的最小值就一定为1咯;后者的话对应的是f[i-j][j],把里面的j个数整体加上1的话，其里面的最小值都大于1咯。

数位统计dp

1，计数问题

emmm,l和1傻傻分不清，导致找了半天错误都没有找出来。

思路：

  • 本题较难的地方在于分类讨论；

  • 针对0-910个数的话，分别去算他们在目标每一位出现的次数；

  • 当所统计的数不在开头位置的话（假设该数为abcdef，现在在统计第三位某一个数出现的次数的话）：

    • 如果所统计的数不是0的话，能取的数的范围为：0-(ab-1),总共有ab个数，所以结果为res += ab * 1000;

    • 如果统计的数是0的话，由于不能出现前导0，所以第一次部分的枚举只能从(01)-(ab-1),所以结果就为res += (ab-1) * 1000;

  • 上面统计的数最大也是到ab-1,所以的话现在需要讨论一下ab时该数出现的次数，假设所统计的数为x：

    • 如果x > c的话, 则无解；

    • 如果x == c的话，res += def+1;

    • 如果x < c的话，res += 1000.

  • 还有一点需要注意的是，如果统计的是0的次数，直接跳到倒数第二位统计就行，前导0的情况是我们应该避免的。

代码实现：O(10*位数*位数)

状压dp

1，蒙德里安的梦想

思路：

  • 这道题的话是给定nxm的矩阵问你有多少种放置1x2木块的方案数，可以转换成横向放置1x2木块的合法方案数，因为你一旦确定横向放置的位置，纵向放置的位置也随之确定了。

  • 放置合法需要注意到两点：

    • 横向放置的相邻间隔为偶数（预处理）；

    • 当前所放置的列数不能跟前一列有重合（状态&）。

  • 初始状态只有f[0][0]=1算是一种合法的方案，其余方案为0。

  • 状态表示：f[i][j]表示在在第i列放置状态为j的木块的方案数为多少；

  • 状态转移：从前一列满足合法条件的状态转移过来就可以。

代码实现：$(m2^n2^n)$

树形dp

1，没有上司的舞会

思路：

  • 初始化，主要是考虑dfs到叶子节点的时候怎样初始化。因为对于叶子节点来说的化，只有取或者不取两种状态，不取的状态对应为0，而取的状态的化，其值对应的是happy[u]；

  • 第二个需要思考的东西是节点有多少种状态，这里的话是2；

  • 状态的表示，因为节点有两种状态，分类讨论下：

    • 以u为根的子树在选择u的情况下所能获得的最大happy值；

    • 以u为根的子树在不选择u的情况下所能获得的最大happy值。

  • 状态转移：如果子树选择了其根节点的话，他能选择的状态只能从以儿子节点为根节点的子树当中选择没有选择儿子节点的状态转移过来；如果子树没有选择其根节点的话，可以由两种状态中较大的那一个转移过来。

代码实现：$O(n)$

记忆化搜索

1,滑雪问题

思路

  • 主要是在搜索的时候记录一下已经计算过的值，等到下次搜索到同样的状态时返回计算过的值就行了。

  • 记忆化搜索的好处在于代码写起来简单明了，相对于写for循环求出所有状态前需要考虑一下各层的循环的顺序，记忆化搜索就显得很无脑了，你需要什么状态就搜索一下并记录下来就行，不用去考虑循环的顺序，代价的话速度慢点（在可以接受的范围），可能会出现爆栈的情况，但是这些情况出现的概率较低。

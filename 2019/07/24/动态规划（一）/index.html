背包问题

背包问题的话之前已经写过关于背包九讲的博客了，在这里的话就不多赘述了。

这里的话总结以下yxc老师提到从集合的角度去思考dp

  • 状态的表示

    • 集合。主要考虑当前状态的所有选择以及对应的条件

    • 属性。主要有三种，max，min，数量。

  • 状态的计算。考虑当前集合的划分。

这个理解角度的话对于一些不熟悉的dp问题很有帮助。

线性dp

1.数字三角形

思路：

  • 状态表示：f[i][j]从(i,j)出发到达末尾的最大值；

  • 状态转移：f[i][j] += max(f[i+1][j], f[i+1][j+1])

代码实现：$O(n^2)$

2,最长上升子序列

思路：

  • 状态表示：f[i]以i结尾的最长上升子序列的长度；

  • 状态转移：f[i] = max(f[i], f[j]+1)(j<i && a[j]<a[i])

另一种思路：

  • 状态表示：f[i]表示的是所有长度为i的最长上升子序列当中结尾的最小值。

  • 状态转移：通过二分查询当前数插入的位置。

代码实现：$O(n^2)$

代码实现：$O(nlogn)$

3，最长公共子序列

思路：

  • 状态表示：f[i][j]第一个序列的前i和第二个序列的前j个字母的最长公共子序列的长度；

  • 状态转移：

    • 如果s[i]==t[j]，f[i][j]=f[i-1][j-1]+1

    • 如果s[i]!=t[j]，f[i][j]=max(f[i-1][j], f[i][j-1])

代码实现：$O(nm)$

4，最短编辑距离

思路

  • 状态表示：f[i][j]第一个字符串的前i个字母变成第一个字符串前j个字母所需的最小代价；

  • 状态转移：修改的话有三种操作：增，删，改。下面的话分类讨论：

    • 如果s[i] == t[j],就不需要修改操作，f[i][j] = f[i-1][j-1];

    • 下面是取三种操作，值较小的一个：

    • 如果是增的话，f[i][j] = f[i][j-1] + 1;

    • 如果是删的话，f[i][j] = f[i-1][j] + 1;

    • 如果是改的话，f[i][j] = f[i-1][j-1] + 1。

代码实现：$O(nm)$

区间dp

看到定义之后想到之前做过的一道打爆气球的题目，那是用记忆化搜索做的。

区间dp整体写法的思路是：先循环区间长度，再循环左端点，最后循环区间的决策。

石子合并

思路：

  • 状态表示：f[i][j]表示合并区间[i,j]石堆所花费的最小代价，做法是枚举区间的最后一次合并的位置以及区间长度从小到大枚举，保证计算f[i][j]时其依赖的状态已经计算出来了；

  • 状态转移：枚举最后合并位置保留最小值。

代码实现：$O(n^3)$

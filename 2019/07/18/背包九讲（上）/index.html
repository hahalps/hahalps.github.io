背包九讲（上）

Veröffentlicht am 2019-07-18

主要是回忆一下自己学过的九种背包以及他们个各自的优化有什么。

01背包

题面：现在有n件物品，每件物品有其对应的体积和价值，针对每件物品，你只能选或者不选，现在给你一个体积为v的背包，问你背包在不超过背包容量的情况下，所能装下的最大价值为多少？

先定义状态：f[i][j], 是说在决策完前i件物品后，背包容量为j的最大价值为多少。

状态转移的话， f[i][j] = max(f[i-1][j], f[i-1][j-v[i]]+w[i]), 针对的就是第i件物品，选或者不选的决策。

因为f[i][j]的状态跟f[i-1][j]和f[i-1][j-v[i]]有关，所以我们可以从大到小，从右到左遍历，把f[i][j]压缩成f[j]就可以了。

对了，补充一个初始化的问题，上面那样初始化的话最终输出的答案所对应的背包的容量可能没有被装满，比如说有这样的一个例子物品= ((1, 1000), (2, 2))，背包体积=2

如果要求的背包恰好装满的话，只需要一开始把除了f[0]之外的都初始为负无穷，除0之外是因为他们的状态不合法且负无穷的话是因为我们求的是最大值，如果最小值的话，那就相反咯。

完全背包

题面：现在同样有n件物品，跟上面的不同的是，每件物品可选择的数量为无限多件，问你在这样的情况下，背包能装下的最大价值为多少？

状态的定义：f[i][j], 决策完前i件物品后，背包容量为j所能装下的最大价值。

状态的转移：f[i][j] = max(f[i-1][j], f[i-1][j-k x v[i]] + k x w[i]), 该方程的意思是往容量为j的背包尝试放0件，1件， 2件……k件， k是背包容量为j所能放入i物品的最大件数。其实针对k-1件前的情况，在f[i][j-v[i]]已经决策过了，所以的话状态转移方程可以写为f[i][j] = max(f[i-1][j], f[i][j-v[i]]+w[i])就可以了

多重背包

题面：多重背包的话，与上面不同的地方在与每件物品是一个给定的数目，针对多重背包的话有三种做法，一是普通做法，二是二进制优化，三是单调队列优化。

状态表示：f[i][j]，决策完前i件物品之后，容量为j的背包所能装下的最大价值。

状态的转移: f[i][j] = max(f[i-1][j], f[i-1][j-k x v[i]] + k x w[i]), k 的取值范围为当前物品数量的最大值和背包容量为j所能装下的最大数量，两者之间取个最小的。

二进制优化的话，分解的是对应物品的数量，分解类似于（7=1+2+4），再跑一遍01背包就可以了。这样可行的原因对于1 2 4这三个数来说，可以拼凑成0-7这八个数，这样的话，就可以不遗漏的考虑针对这件物品取0-k件的情况了。这里的话有一个容易犯错的点在于你是分解了n件物品的数量，分解完的总件数应该等于nlogm。

单调队列优化听说是男人八题的一道，hhh。其实的话，核心的做法在于维护一个长度为k的滑动窗口，维护的是一个单调递减的窗口。之前拎不清楚的地方针对滑动窗口里面的两个数是通过什么样的方式判断其大小的，对了，刚想起来，对于第i个物品来说分成v类，这样分的的原因也很简单，因为对于f[i][j],他所需要知道的数无非是f[i-1][j-v[i]], f[i-1][j-2*v[i]], ……f[i-1][j-k*v[i]]这些数，其他的数值我是不需要知道的，所以才可以分成把物品i分成v类。emm,单调队列里面判断大小是怎么判断的呢？假设该功能实现成一个函数cmp(),呀呀,短路了。

举个例子来说吧，满脑子空想也不是办法。对于f[i][j]来说，他需要的是f[i-1][j], f[i-1][j-v[i]]+w[i], f[i-1][j-2*v[i]]+2*w[i]……j[i-1][j-l*v[i]]+k*w[i], 而对于f[i][j+v]来说的是，他需要的是f[i-1][j+v], f[i-1][j]+w[i], f[i-1][j-v[i]]+2*w[i]……j[i-1][j-(k-1)*v[i]]+k*v[i]。

再从中挑出一个位置来说吧:比如说f[i-1][j], 对于f[i][j+v]来说的，需要的是他加上w的值，对于f[i][j+2v]来说的话，需要的是他加上2w的值，以此类推…对于说f[i-1][j-v], 对于f[i][j+v]来说的，需要的是他加上2*w的值，对于f[i][j+2v]来说的话，需要的是他加上3w的值，以此类推…

没有一种直观的感受，再想想吧。。。一边写一边想。

其实单调队列直接按照原值存的话，求下一次所有进行的操作是判断队头是否过期，且需要对单调队列的整体加上一个w, 这个其实就是我们需要优化的地方，说实话，我们其实要比较的是每个数的潜力值，所以的话我要把他们移到“同一起跑线上”进行比较,也就是移到开头的位置上进行比较，移的方式为？？？ ，哦哦哦，懂了。

爽啊啊啊啊啊啊啊啊啊，无比的舒畅。。

二维背包的费用问题

二维背包的费用问题其实是相对于比上面多出了一个限制，上面限制背包所能装的最大数量的因素在于背包的体积，二维的话，就是在这个的基础上多出了重量的因素，（其实按照这种情况的话，我们可以把二维的情况扩大到n维的情况），针对与这个情况的话，可以考虑这样去做:第一层的循环循环物品，从第二层开始循环各种限制就可以了。
